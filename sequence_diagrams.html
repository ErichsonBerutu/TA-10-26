📊 Sequence Diagram UML — Aplikasi Mobile

Sistem Administrasi Desa Hutabulu Mejan (Gaya Robustness / Analisis UML Asli)

📌 Informasi Akun: Akun masyarakat (NIK & Password) didaftarkan dan diberikan oleh Perangkat Desa melalui sistem web admin. Tidak ada fitur registrasi mandiri di aplikasi mobile demi keamanan data kependudukan.

1. Sequence Diagram Login

Proses masuk masyarakat menggunakan NIK dan password dari perangkat desa. LoginPage memvalidasi format input (NIK harus 16 digit), lalu mengirim data ke AuthService untuk diverifikasi dengan server Laravel API.

2. Sequence Diagram Logout

Proses keluar dari akun secara aman. Sistem menghapus token FCM dari Firebase Messaging, menghapus data login dan token kependudukan dari SharedPreferences lokal, serta mengirim notifikasi logout asinkron ke server Laravel API.

3. Sequence Diagram Pengajuan Surat

Masyarakat memilih jenis surat, melihat persyaratan dinamis yang diambil dari database/API, mengisi formulir dengan data valid, dan mengirimkannya ke server. Jika offline, data otomatis dialihkan ke sync queue lokal.

4. Sequence Diagram Pengaduan Masyarakat

Masyarakat membuat laporan pengaduan berupa teks dan foto. PengaduanService menyimpan data offline jika koneksi internet terputus, dan mengirimkannya saat perangkat kembali online.

5. Sequence Diagram Melihat Berita

Masyarakat membaca berita desa. Sistem memprioritaskan database lokal (local-first) agar berita dapat diakses offline secara instan, lalu menyegarkan data dari server di latar belakang.

6. Sequence Diagram Melihat Pengumuman

Proses pemuatan daftar pengumuman desa. Aplikasi menggunakan database cache SharedPreferences agar pengumuman langsung termuat, kemudian melakukan request data terbaru ke server secara asinkron.

7. Sequence Diagram Notifikasi

Sistem notifikasi real-time berbasis Firebase Cloud Messaging (FCM). FCM Token dikirim ke server saat login, dan NotifikasiService melakukan polling periodik serta menandai status baca notifikasi.

8. Sequence Diagram Download PDF Surat

Masyarakat dapat mengunduh surat resmi yang telah disetujui perangkat desa. Surat diunduh langsung dari server desa dalam format PDF lalu dicetak/disimpan ke memori internal menggunakan Printing Plugin.

9. Sequence Diagram Riwayat Layanan

Masyarakat memantau riwayat pengajuan surat. PengajuanService memadukan data dari sync queue (proses offline) diletakkan paling atas, lalu digabung dengan data cache lokal dan data server API.

10. Sequence Diagram Lihat Profil

Melihat profil terdaftar. Profil lokal termuat terlebih dahulu, kemudian AuthService menyegarkan data profil terbaru (seperti nama, alamat, status nikah, dll.) dari server secara background.

11. Sequence Diagram Respons Admin

Masyarakat berkomunikasi dengan admin/perangkat desa terkait status pengajuan atau pengaduan secara dua arah. Pesan balasan dikirim langsung ke server Laravel, dan riwayat pesan di-cache secara lokal.
