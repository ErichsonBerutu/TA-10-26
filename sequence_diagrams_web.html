1. Sequence Diagram Login & Autentikasi Admin

Proses masuk pengguna dengan hak akses admin/staff ke web dashboard. Sistem memvalidasi kredensial dan melakukan pengecekan middleware role. 
sequenceDiagram
    autonumber
    actor A as "Admin / Perangkat Desa"
    participant LV as "LoginView (Blade)"
    participant AC as AuthController
    participant MW as "Auth Middleware"
    participant DB as "Database (Tabel: users)"

    A->>LV: Buka halaman login web (/login)
    activate LV
    LV-->>A: Tampilkan form login (Email dan Password)
    deactivate LV
    A->>LV: Isi form login dan klik Login
    activate LV
    LV->>AC: POST /login (credentials)
    activate AC
    AC->>DB: Cek kecocokan email dan hash password
    activate DB
    DB-->>AC: return user object dan role ('admin' / 'staff')
    deactivate DB
    
    alt Kredensial Valid dan Role Sesuai
        AC->>AC: Buat Session Login (Auth::login)
        AC-->>LV: Redirect ke halaman Dashboard Admin
        LV->>MW: Akses route /admin (Dashboard)
        activate MW
        MW->>MW: Validasi role dan session aktif
        MW-->>LV: Izinkan akses (200 OK)
        deactivate MW
        LV-->>A: Tampilkan Halaman Dashboard Admin
    else Kredensial Tidak Valid
        AC-->>LV: Kembalikan error Email/password salah
        LV-->>A: Tampilkan pesan error pada Form Login
    end
    deactivate AC
    deactivate LV

2. Sequence Diagram Tambah Data Penduduk

Admin memasukkan data kependudukan baru (Induk Penduduk) ke database desa via controller Livewire, termasuk validasi unik NIK 16 digit. 
sequenceDiagram
    autonumber
    actor A as Admin
    participant IV as "IndukPendudukCreateView (Livewire)"
    participant IC as IndukPendudukCreateController
    participant DB as "Database (Tabel: penduduk)"

    A->>IV: Buka Halaman Tambah Penduduk
    activate IV
    IV-->>A: Tampilkan form data diri (NIK, Nama, dll.)
    deactivate IV
    A->>IV: Isi data lengkap dan klik Simpan
    activate IV
    IV->>IC: Panggil fungsi store(Request)
    activate IC
    
    IC->>IC: Validasi Input (NIK wajib 16 digit, required fields)
    IC->>DB: Query Cek duplikasi NIK di tabel penduduk
    activate DB
    DB-->>IC: return status (NIK terdaftar / belum)
    deactivate DB

    alt NIK Belum Terdaftar (Valid)
        IC->>DB: Insert data baru ke tabel penduduk
        activate DB
        DB-->>IC: return success
        deactivate DB
        IC-->>IV: Redirect ke index dengan flash message sukses
        IV-->>A: Tampilkan list penduduk terupdate dan alert sukses
    else NIK Sudah Terdaftar (Duplikasi)
        IC-->>IV: Kembalikan validation error (NIK sudah terdaftar)
        IV-->>A: Tampilkan form kembali dengan highlight error NIK
    end
    deactivate IC
    deactivate IV

3. Sequence Diagram Tambah Akun Penduduk (Registrasi Mobile)

Admin mendaftarkan akun login mobile untuk warga yang sudah terdaftar di data penduduk. Sistem membuat credential, menyimpannya di DB users, dan mencatatnya ke log file spreadsheet CSV local secara otomatis. 
sequenceDiagram
    autonumber
    actor A as Admin
    participant IV as "IndukPendudukView (Livewire)"
    participant IC as IndukPendudukController
    participant DB as Database
    participant FS as "FileSystem (CSV Log)"

    A->>IV: Klik Buat Akun pada data penduduk pilihan
    activate IV
    IV->>IC: openCreateAccountModal(id_penduduk)
    activate IC
    IC->>DB: Query data nama dan NIK penduduk
    activate DB
    DB-->>IC: return penduduk object
    deactivate DB
    IC-->>IV: Tampilkan Modal Form Tambah Akun Warga
    deactivate IC
    deactivate IV

    A->>IV: Masukkan Email, Password dan Konfirmasi Password
    activate IV
    A->>IV: Klik tombol Simpan Akun
    IV->>IC: Panggil fungsi saveAccount()
    activate IC
    IC->>IC: Jalankan validasi (Email unik, Password min 6)
    
    IC->>DB: Insert data baru ke tabel users (role='masyarakat', id_penduduk, password_plain)
    activate DB
    DB-->>IC: return success
    deactivate DB

    Note over IC,FS: Proses Pencatatan Spreadsheets CSV
    IC->>FS: Buka berkas public/excel/pertinggal_akun_warga.csv
    activate FS
    FS->>FS: Tambahkan baris baru (Nama, NIK, Email, Password Polos, Tanggal Dibuat)
    FS-->>IC: return success
    deactivate FS

    IC-->>IV: Dispatch SweetAlert sukses (Akun Berhasil Dibuat!)
    deactivate IC
    activate IV
    IV-->>A: Tutup modal dan tampilkan alert sukses pembuatan akun
    deactivate IV

4. Sequence Diagram Manajemen Akun (Soft Delete & Force Logout)

Alur admin menonaktifkan akun warga secara temporer. Sistem menghapus token otentikasi Sanctum (memaksa warga logout otomatis di HP) dan melakukan soft-delete pada akun di database. 
sequenceDiagram
    autonumber
    actor A as Admin
    participant MV as "ManajemenAkunView (Livewire)"
    participant MC as ManajemenAkunController
    participant DB as "Database (Tabel: users)"

    A->>MV: Klik Nonaktifkan Akun pada baris warga
    activate MV
    MV->>MC: confirmDelete(id)
    activate MC
    MC->>DB: Hitung total riwayat pengajuan dan pengaduan
    activate DB
    DB-->>MC: return count pengajuan dan pengaduan
    deactivate DB
    MC-->>MV: Dispatch SweetAlert konfirmasi dengan info dampak nonaktif
    deactivate MC
    deactivate MV

    A->>MV: Klik tombol konfirmasi Ya Nonaktifkan
    activate MV
    MV->>MC: Panggil fungsi delete()
    activate MC
    
    Note over MC,DB: Database Transaction Start
    MC->>DB: Hapus semua Sanctum token warga (personal_access_tokens)
    activate DB
    DB-->>MC: return success
    deactivate DB

    MC->>DB: Lakukan soft-delete pada user (isi field deleted_at)
    activate DB
    DB-->>MC: return success
    deactivate DB
    Note over MC,DB: Database Transaction End

    MC-->>MV: Redirect/Flash session success
    deactivate MC
    activate MV
    MV-->>A: Tampilan terupdate dengan status akun Tidak Aktif
    deactivate MV

    autonumber

6. Sequence Diagram Pencarian & Filter Pengajuan Surat

Bagaimana Admin mengelola ribuan data pengajuan surat masuk. Controller melayani filter dinamis berdasarkan status, jenis surat, maupun pencarian teks nama/NIK. 
        sequenceDiagram
            autonumber
            actor A as Admin
            participant RV as "RequestSuratIndexView (Blade)"
            participant RC as RequestSuratController
            participant DB as "Database (Tabel: pengajuan_surat)"

            A->>RV: Buka halaman Permintaan Surat (/admin/layanan-surat/request)

            RV->>RC: GET index() dengan parameter (search, status, jenis)
            activate RC

                RC->>DB: where (nama_lengkap / NIK / nama_surat LIKE search)

            alt Parameter 'status' terisi
                RC->>DB: where (status = status)

            alt Parameter 'jenis' terisi
                RC->>DB: where (id_jenis_surat = jenis)

            DB-->>RC: return paginated data dan stats count
            deactivate DB

            RC-->>RV: Render view 'request-surat-controller' dengan data list dan statistik
            deactivate RC
            RV-->>A: Tampilkan dashboard tabel pengajuan surat beserta badge total statistik

7. Sequence Diagram Verifikasi Pengajuan (Setujui & Auto-PDF)

Persetujuan surat oleh admin yang secara otomatis merender HTML ke PDF resmi melalui DomPDF, menyimpannya di storage, dan mengirim push notification ke perangkat warga via Firebase. 
sequenceDiagram
    autonumber
    actor A as Admin
    participant SV as "RequestSuratShowView (Blade)"
    participant RC as RequestSuratController
    participant DB as Database
    participant DOM as "DomPDF Facade (Barryvdh)"
    participant ST as "Storage Server"
    participant FCM as "FcmService (Firebase)"
    actor M as "Aplikasi Mobile Warga"

    A->>SV: Buka detail pengajuan surat masuk
    activate SV
    SV-->>A: Tampilkan data formulir dan berkas persyaratan
    deactivate SV
    A->>SV: Input nomor surat resmi dan klik tombol Setujui
    activate SV
    SV->>RC: PATCH /layanan-surat/request/{id}/setujui (nomor_surat)
    activate RC

    RC->>DB: Cari Kepala Desa yang berstatus aktif (is_active = true)
    activate DB
    DB-->>RC: return data kepala_desa (nama dan NIP)
    deactivate DB

    Note over RC,DOM: Proses Auto-Generate PDF Surat Resmi
    RC->>RC: Render view 'print-surat' menjadi string HTML
    RC->>DOM: loadHTML(html)
    activate DOM
    DOM->>DOM: Set format paper ('a4', 'portrait')
    DOM-->>RC: return PDF binary object
    deactivate DOM

    RC->>ST: Simpan file PDF (storage_path('app/surat/YYYYMMDD_id.pdf'))
    activate ST
    ST-->>RC: return relative filePath
    deactivate ST

    RC->>DB: Update status='selesai', file_pdf=filePath, id_diproses_oleh=idKades
    activate DB
    DB-->>RC: return success
    deactivate DB

    Note over RC,FCM: Proses Notifikasi ke Warga
    RC->>DB: Simpan record ke tabel notifikasi (user_id, judul, pesan)
    activate DB
    DB-->>RC: return success
    deactivate DB
    
    RC->>FCM: sendToUser(user, judul, pesan)
    activate FCM
    FCM->>M: Kirim Firebase Cloud Message (FCM Token) ke HP warga
    FCM-->>RC: status sent
    deactivate FCM

    RC-->>SV: Redirect back dengan flash message sukses
    deactivate RC
    activate SV
    SV-->>A: Tampilkan status surat diperbarui (Selesai) dan alert sukses
    deactivate SV

9. Sequence Diagram Pengaduan Masyarakat (Catatan Admin)

Bagaimana Admin meninjau laporan keluhan/pengaduan masyarakat, otomatis menandainya sebagai 'diproses' setelah dibaca, dan memberikan tanggapan resmi (Catatan Admin) untuk kemudian diselesaikan atau ditolak. 
sequenceDiagram
    autonumber
    actor A as Admin
    participant PV as "PengaduanView (Livewire)"
    participant PC as PengaduanSuratController
    participant DB as Database

    A->>PV: Buka halaman kelola Pengaduan
    activate PV
    A->>PV: Klik tombol Lihat Detail pada salah satu aduan
    PV->>PC: Panggil fungsi showDetail(id)
    activate PC
    
    PC->>DB: Query pengaduan beserta data user pengirim
    activate DB
    DB-->>PC: return pengaduan object
    deactivate DB

    alt Status pengaduan = 'baru'
        PC->>DB: Update status = 'diproses' (Tandai terbaca)
        activate DB
        DB-->>PC: return success
        deactivate DB
    end

    PC-->>PV: Buka modal detail dan muat catatan_admin lama
    deactivate PC
    PV-->>A: Tampilkan isi lengkap aduan, bukti foto dan form respon
    deactivate PV

    A->>PV: Tulis umpan balik di kolom Catatan Admin
    activate PV
    A->>PV: Klik tombol Selesai atau Tolak
    PV->>PC: simpanCatatan(status)
    activate PC
    
    PC->>DB: Simpan perubahan status dan catatan_admin ke DB
    activate DB
    DB-->>PC: return success
    deactivate DB

    PC-->>PV: Flash session success dan closeDetailModal()
    deactivate PC
    activate PV
    PV-->>A: Tutup modal detail dan segarkan daftar aduan
